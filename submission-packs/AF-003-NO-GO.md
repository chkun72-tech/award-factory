# AF-003｜NO-GO Decision

Date: 2026-07-16

Decision: **Do not continue AF-003 as the primary weekly submission.**

## Reason

Official Devpost requirements include a hard blocker:

- Proof of Alibaba Cloud backend deployment is required.
- The proof must be visible through code/repository evidence.
- A public demo video is also required.
- Repository must be public and open source with a visible license.

The current team setup does not have a ready Alibaba Cloud deployment path. Continuing to force this submission would waste time.

## What can be reused

The AF-003 work is not deleted. It can be reused as:

- demo narrative,
- quote-to-follow-up workflow concept,
- 3-minute video structure,
- OpenAI Build Week submission material,
- future Alibaba Cloud submission if the deployment blocker is solved.

## Replacement recommendation

Move primary weekly effort to:

```text
AF-002 OpenAI Build Week
```

Why:

- Australia is listed as eligible.
- No Alibaba Cloud deployment proof requirement.
- Existing Award Factory / Codex workflow itself is a good fit.
- Current repo already demonstrates Codex-built product workflow.
- Required deliverables are more reachable:
  - working project,
  - project description,
  - <3-minute public YouTube video,
  - code repository,
  - README/setup instructions,
  - Codex session/feedback evidence.

## Lesson

Before preparing assets for any hackathon, verify official hard requirements first:

1. eligibility,
2. required platform/cloud,
3. repo visibility/licence,
4. video requirements,
5. deployment proof,
6. deadline/timezone.
