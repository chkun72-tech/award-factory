# Simple Submission Strategy｜低門檻投稿策略

Date: 2026-07-16

## Decision

Do **not** make GitHub-heavy global hackathons the main submission lane right now.

Reason:

- Qwen Cloud requires Alibaba Cloud deployment proof.
- OpenAI Build Week is feasible but crowded and still needs a public video, repository polish, and strong Codex/GPT story.
- Competing directly with global expert builders is not the best first move.

## New main lane

Focus on simpler, slower, reusable submission assets:

1. Startup Award Australia 2026
2. CSIRO Kick-Start
3. NSW MVP Ventures next round / watchlist
4. SXSW Pitch only if the pitch deck and traction are strong enough

## Why this is better

These directions reward:

- clear product story,
- real customer problem,
- market and traction evidence,
- commercialisation plan,
- founder/company readiness,
- Australian relevance.

They do not require winning a global coding contest against advanced hackathon teams.

## What to build next

Create one reusable **Submission Core Pack**:

- one-line product description,
- 150-word product summary,
- 500-word founder/product story,
- problem / solution / market / traction sections,
- current prototype screenshots,
- traction sheet,
- pitch deck outline,
- evidence checklist,
- eligibility documents checklist.

## Recommended first target

AF-015 Startup Award Australia 2026

Why:

- Deadline is later.
- More time to prepare.
- It matches Australian business/product positioning.
- It can reuse the same pitch materials for future grants and awards.

## Parking Lot

Keep these, but do not make them the main lane:

- AF-002 OpenAI Build Week
- AF-003 Qwen Cloud
- AF-005 CockroachDB/AWS
- AF-006 DataHub

They are useful for visibility and practice, but not the safest first submission path.

## Immediate next step

Prepare the reusable Submission Core Pack for Handyman AI / Award Factory positioning.
