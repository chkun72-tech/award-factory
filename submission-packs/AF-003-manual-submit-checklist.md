# AF-003｜Manual Submit Checklist

Use this before submitting to Devpost.

## 1. Official rule verification

- [ ] Confirm Australia-based participant is eligible.
- [ ] Confirm whether the submission must use Qwen.
- [ ] Confirm whether Alibaba Cloud usage is required or optional.
- [ ] Confirm whether GitHub repo must be public.
- [ ] Confirm whether demo video must be public or unlisted.
- [ ] Confirm IP/licence terms allow later commercial use.
- [ ] Confirm final deadline in Australia/Sydney time.

## 2. Demo readiness

- [ ] Demo opens without install errors.
- [ ] Customer enquiry is visible.
- [ ] `Run Autopilot` reveals all output cards.
- [ ] Output includes job scope.
- [ ] Output includes missing-information questions.
- [ ] Output includes quote draft.
- [ ] Output includes customer reply draft.
- [ ] Output includes follow-up task.
- [ ] Human approval boundary is visible.

## 3. GitHub readiness

- [ ] Repo exists.
- [ ] Code pushed.
- [ ] README explains problem, workflow, and demo.
- [ ] No `.env` or secrets committed.
- [ ] If required by rules, repo visibility is public before submission.
- [ ] If public, no private customer data or credentials included.

## 4. Video readiness

- [ ] Video is under 3 minutes.
- [ ] Font size is readable.
- [ ] Shows before/after clearly.
- [ ] Mentions Qwen/Alibaba Cloud integration path if required.
- [ ] Mentions human approval boundary.
- [ ] Does not show private data.

## 5. Devpost fields

- [ ] Project title pasted.
- [ ] Tagline pasted.
- [ ] Inspiration pasted.
- [ ] What it does pasted.
- [ ] How we built it pasted.
- [ ] Challenges pasted.
- [ ] Accomplishments pasted.
- [ ] What we learned pasted.
- [ ] What's next pasted.
- [ ] Built with tags selected.
- [ ] GitHub URL added.
- [ ] Demo video URL added.
- [ ] Screenshot uploaded.

## Go / no-go

Submit only if:

- eligibility is confirmed,
- demo works,
- no secrets are exposed,
- video and repo links are accessible,
- human approval boundary is clear.
